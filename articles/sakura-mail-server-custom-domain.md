## 0. 背景

自分が協力している会社で複数の独自ドメインを利用してサイトを公開しているのですが、それぞれの独自ドメインでメールを利用する必要があり、容量や料金、ドメイン利用可能数等を比較したところ、[さくらのレンタルサーバのメールボックスプラン](https://rs.sakura.ad.jp/plan/mail/)が最も良い選択肢であったため今回利用しました。
Google WorkspaceのStarterプランも比較対象でしたが、利用人数が増えるとその分月額料金も大きくなり、かつGeminiやMeetなどのサービスも現状利用していないため今回は見送りました。
~~Cloudflareにメール送信機能が早く実装されればいいなぁ~~

## 1. 概要

Cloudflareなどで独自ドメインを利用してサイトを公開しても、独自ドメインでのメールの受信はCloudflare Email Routingを利用すれば可能ですが送信は現状(2025/12/29)ではできません。
そのため別途メールサーバーを借りてメールを送信する必要があります。もちろん、GmailのSend as機能によって独自ドメインによる送信も不可能ではありませんが、スパム判定されかねないため、企業として利用する方法としては適切ではありません。
ということで今回はさくらのレンタルサーバを利用して独自ドメインでメールを送信する方法をご紹介します。

## 2. さくらのレンタルサーバーでメールサーバーの契約

https://rs.sakura.ad.jp/plan/mail/ にアクセスしてプランを選択してください。

![](/images/sakura-mail-server-custom-domain/2025-12-29-01-25-44.png)

今回はメールボックスプランを選択しました。
するとお申し込み画面が表示されるので、上から順に、
- 希望プラン
  - 今回はさくらのメールボックス
- 初期ドメイン
  - 後から変更できるため適当で良い
- 独自ドメイン
  - 持っていない人は独自ドメインをここで取得しても良いですが、今回は取得済みのためレンタルサーバだけの契約をしました

の選択状態を確認して、`お支払い方法の選択`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-01-31-39.png)

すると`さくらインターネット 会員認証`画面が表示されるので、そこで新規会員登録またはログインを行ったあと、お支払いしてレンタルサーバーの契約が完了します。

## 4. 独自ドメインの設定

次に会員メニューにログインし、`契約中のサービス一覧`から`コントロールパネルを開く`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-01-47-42.png)

すると`サーバーコントロールパネル ホーム`画面が表示されるので、画面左のサイドバーから、`ドメイン/SSL`→`ドメイン/SSL`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-02-07-48.png)

そして`ドメイン新規追加`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-02-14-13.png)

すると複数の選択肢が出てくるので、今回は`他社で取得したドメインを移管せずに使う`を選択します。
この選択肢にした理由として、さくらインターネットを利用するのはメールサーバーのみであるため、特に移管する理由がないためです。
逆にさくらインターネットのサービスを他にも利用する場合は移管しても良いかもしれません。

![](/images/sakura-mail-server-custom-domain/2025-12-29-02-18-42.png)

その後登録したいドメインを入力し、`追加`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-02-42-35.png)

## 5. DNSレコードの設定

先ほどの画像の下部にネームサーバーを変更する旨のメッセージが表示されていますが、今回は`.jp`ドメインをお名前.comで取得して、ネームサーバーをCloudfalreに登録しているので行えません。
そのためここではCloudflareのDNSレコードを設定します。

### 5.1. MXレコードの設定

| 項目 | 値 |
| ---- | ---- |
| タイプ | MX |
| 名前 | @ |
| 優先度 | 10 |
| メールサーバー | [コントロールパネルのサーバー情報にあるホスト名] |
| プロキシステータス | DNSのみ |

### 5.2. SPFレコードの設定

| 項目 | 値 |
| ---- | ---- |
| タイプ | TXT |
| 名前 | @ |
| コンテンツ | v=spf1 [コントロールパネルのサーバー情報にあるホスト名] mx ~all |

### 5.3. DKIMレコードの設定

メール内容が改ざんされていないことを証明するためのレコードです。

| 項目 | 値 |
| ---- | ---- |
| タイプ | TXT |
| 名前 |  [セレクタ]._domainkey |
| コンテンツ | さくらのコントロールパネルで取得した公開鍵 |
| プロキシステータス | DNSのみ |

コンテンツはコントロールパネルの`ドメイン/SSL`ページ内の先ほど追加した独自ドメインの`設定`から`DKI/ARC設定`を選択してください。

![](/images/sakura-mail-server-custom-domain/2025-12-29-03-20-28.png)

すると`DKIM/ARC設定`が表示され、デフォルトでは`DKIMレコード`がONになっているのですが、今回Cloudflareで設定するためOFFにします。
そして`設定する`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-03-29-33.png)

`公開鍵`が表示されるのでコピーし、CloudflareのDNSレコードのコンテンツに`v=DKIM1; k=rsa; p=[公開鍵]`の形式で設定します。

![](/images/sakura-mail-server-custom-domain/2025-12-29-03-45-55.png)

### 5.4. DMARCレコードの設定（オプション）

| 項目 | 値 |
| ---- | ---- |
| タイプ | TXT |
| 名前 | _dmarc |
| コンテンツ | v=DMARC1; p=none; adkim=s; aspf=s; rua=mailto:dmarc@example.com |

ここの`rua=mailto:dmarc@example.com`は存在するメールアドレスにしてください。
今回追加する独自ドメインのメールアドレスで構いませんが、認証結果の統計や送信件数などのレポートが届くため確認できるようにしておきます。

最終的に以下のようにCloudflareのDNSレコードに追加で4つされていれば設定完了です。

![](/images/sakura-mail-server-custom-domain/2025-12-30-01-12-40.png)

## 6. メールアカウントの作成

コントロールパネルの`メール`→`メールアドレス`を選択します。

![](/images/sakura-mail-server-custom-domain/2025-12-29-04-04-10.png)

そして`新規追加`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-29-04-18-41.png)

`基本項目`を埋めます。

ユーザー名に関しては先ほどのruaのmailto:に指定したアドレスをユーザー名として利用します。

![](/images/sakura-mail-server-custom-domain/2025-12-29-04-10-02.png)

`メール利用範囲`を埋めます。
以下の設定で作成しました。

- メール容量制限
  - そこまで大量に利用するアドレスではない想定なのでデフォルトは1GBでしたが、今回100MBにしました。
  - 今回選択したプランだと最大20GBまでだったので、それ以上の指定はできません。
- メールの受信
  - 受信する
- 転送先アドレス
  - メールの受信にて転送専用を選択した場合は必須です。
  - 転送専用でなくとも、Gmailでの利用を想定している方は利用するGmailアドレスを転送先アドレスとして設定しておきましょう。
- 拒否リスト
- 許可リスト
  - 上記2項目は正規表現も利用できるので便利ですね。
- ウイルスチェック
  - 有効が推奨らしいので有効にしておきます。
- 迷惑メールフィルター
  - 簡易迷惑メールフィルタを利用する

![](/images/sakura-mail-server-custom-domain/2025-12-29-04-11-16.png)

## 7. DNS設定の反映確認

コマンドラインで確認してみましょう。

```:terminal
  # MXレコードの確認
  dig MX example.jp
  
  # SPFレコードの確認
  dig TXT example.jp
  
  # DKIMレコードの確認
  dig TXT sakura._domainkey.example.jp
  
  # DMARCレコードの確認
  dig TXT _dmarc.example.jp
```

正常なレスポンスが返ってきていれば反映完了です。

## 10. Gmailでの利用方法

メールアプリよりもGmailでの利用が多い方も多いと思いますので、その方法をご紹介します。

[6. メールアカウントの作成](#6-メールアカウントの作成)で紹介した`転送先アドレス`を利用したいGmailアドレスに設定しておきます。
するとそのアドレスに対してメールが届くようになります。

次にGmailのSend as機能を利用します。

Gmailを開いて左上の歯車アイコンをクリックし、`すべての設定を表示`をクリックすると以下の画像のような設定画面が出てきます。
そこで`アカウントとインポート`を選択し、`名前`蘭の`他のメールアドレスを追加`をクリックします。

![](/images/sakura-mail-server-custom-domain/2025-12-30-00-44-30.png)

そこで名前と今回作成したメールアドレスを入力します。
エイリアスとして扱うことで既存のGmailアドレスのメールボックスでも確認できるようになります。

![](/images/sakura-mail-server-custom-domain/2025-12-30-00-46-44.png)

そしてSMTPサーバー用のユーザー名やパスワードを入力したらアカウントを追加します。

すると、`Gmail からのご確認 - admin@example.jp を差出人としてメールを送信します`という件名のメールが届きます。
そこに記載されている確認リンクをクリックし、画面が遷移したら確認ボタンをクリックするとアカウントが追加されているはずです。

![](/images/sakura-mail-server-custom-domain/2025-12-30-00-56-52.png)

差出人として今回追加したメールアドレスが利用できていれば完了です。

![](/images/sakura-mail-server-custom-domain/2025-12-30-00-59-15.png)

受信も確認できました。

![](/images/sakura-mail-server-custom-domain/2025-12-30-01-02-14.png)
